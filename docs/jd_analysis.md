## 二、核心技能要求

### 2.1 编程语言

| 语言 | 要求程度 | 使用场景 |
|------|---------|---------|
| Python | **必须** | Agent 开发、模型调用、数据处理 |
| TypeScript/JavaScript | 加分 | 前端交互、Web Agent |
| Go/Rust | 加分 | 高性能服务、基础设施 |
| SQL | 必须 | 数据查询、数据库操作 |

### 2.2 AI/ML 基础

| 技能 | 要求程度 | 说明 |
|------|---------|------|
| LLM 原理与应用 | **必须** | 理解 Transformer、Attention、Prompt Engineering |
| RAG 技术 | **必须** | 检索增强生成、向量数据库 |
| Embedding 模型 | **必须** | 文本向量化、语义相似度 |
| Fine-tuning | 加分 | 模型微调、LoRA、QLoRA |
| 多模态模型 | 加分 | 图文理解、语音处理 |

### 2.3 Agent 框架与工具

| 框架/工具 | 要求程度 | 说明 |
|-----------|---------|------|
| LangChain | **必须** | Agent 编排、工具调用 |
| LlamaIndex | **必须** | RAG 索引、数据连接 |
| AutoGen/CrewAI | 加分 | 多 Agent 协作 |
| LangGraph | 加分 | 状态机 Agent |
| MCP (Model Context Protocol) | 趋势 | 工具协议标准 |

### 2.4 数据工程

| 技能 | 要求程度 | 说明 |
|------|---------|------|
| 向量数据库 | **必须** | Chroma、Qdrant、Pinecone、Milvus |
| SQL 数据库 | **必须** | PostgreSQL、MySQL、SQLite |
| NoSQL 数据库 | 加分 | MongoDB、Redis |
| 数据管道 | 加分 | ETL、数据清洗、流处理 |
| 知识图谱 | 加分 | 图数据库、关系抽取 |

### 2.5 基础设施

| 技能 | 要求程度 | 说明 |
|------|---------|------|
| Docker/K8s | **必须** | 容器化部署 |
| 云服务 | **必须** | AWS/Azure/GCP |
| CI/CD | 加分 | GitHub Actions、GitLab CI |
| 监控观测 | 加分 | OpenTelemetry、Prometheus |

## 三、典型 JD 示例

### 3.1 AI Agent 开发工程师（中级）

**岗位职责**：
1. 负责 AI Agent 应用的设计与开发
2. 实现 Agent 与外部系统的集成（API、数据库、文件系统）
3. 优化 Agent 的推理效率和响应质量
4. 设计和实现 Agent 的记忆和上下文管理

**任职要求**：
1. 3 年以上 Python 开发经验
2. 熟悉 LangChain/LlamaIndex 等 Agent 框架
3. 了解 RAG 技术和向量数据库
4. 有 SQL 数据库操作经验
5. 良好的代码质量和工程实践

### 3.2 AI 平台工程师（高级）

**岗位职责**：
1. 设计和构建 AI Agent 开发平台
2. 实现 Agent 的可观测性和监控系统
3. 优化 Agent 基础设施的性能和可靠性
4. 推动 Agent 技术标准化和最佳实践

**任职要求**：
1. 5 年以上后端开发经验
2. 精通 Python，熟悉 Go/Rust
3. 深入理解 Agent 架构和设计模式
4. 有大规模分布式系统经验
5. 熟悉云原生技术和容器编排

### 3.3 AI 数据工程师

**岗位职责**：
1. 设计和实现 Agent 数据管道
2. 构建和维护向量数据库集群
3. 优化数据检索和查询性能
4. 实现数据质量和治理机制

**任职要求**：
1. 3 年以上数据工程经验
2. 精通 SQL 和数据建模
3. 熟悉向量数据库和 Embedding 模型
4. 有 ETL 和数据管道开发经验
5. 了解数据安全和隐私保护

## 六、与我们项目的关联

### 6.1 我们项目覆盖的 JD 需求

| JD 需求 | 我们的覆盖 | 差距 |
|---------|-----------|------|
| 多数据源集成 | ✅ SQL、向量、文件 | 需要补充 API 连接器 |
| Agent 应用开发 | ✅ AgentDataClient | 需要更多示例 |
| 向量数据库 | ✅ InMemoryVector | 需要真实向量数据库连接器 |
| 数据管道 | 🔜 基础实现 | 需要完善 ETL 功能 |
| 可观测性 | ✅ 基础追踪 | 需要集成 OpenTelemetry |
| 缓存优化 | ✅ 上下文感知缓存 | 需要更多缓存策略 |

### 6.2 需要补充的功能

**P0 - 必须实现**：
1. PostgreSQL/MySQL 真实连接器
2. Chroma/Qdrant 向量数据库连接器
3. REST API 连接器
4. 更完整的文档和示例

**P1 - 应该实现**：
1. OpenTelemetry 集成
2. 更多缓存策略（LRU、LFU）
3. 连接池管理
4. 重试和熔断机制

**P2 - 可以考虑**：
1. MCP 协议支持
2. 多 Agent 协作支持
3. 实时流处理
4. 可视化配置界面
