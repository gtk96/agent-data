# AI Agent 工程师 JD 需求分析（基于行业调研）

由于 Boss 直聘、猎聘、拉勾等招聘平台均有强反爬虫机制（WAF + JS 渲染 + 验证码），无法直接抓取 JD。以下分析基于：

1. 头条搜索获取的 AI Agent 招聘趋势文章
2. 行业公开信息和技术社区讨论
3. GitHub 上的 AI Agent 项目和技能要求

## 二、核心发现

> **"AI时代三个你从没听过的新岗位"**
> 两个职位——**Agent Engineer** 和 **Forward Deployed Engineer**。正在从AI和企业的衔接地带长出来。Agent Engineer: 给AI当"班长"

> **"互联网企业释放超20万岗位 'AI+'人才需求攀升"**

## 三、典型 JD 示例

### 3.1 AI Agent 开发工程师（中级）

**岗位职责**：
1. 负责 AI Agent 应用的设计与开发
2. 实现 Agent 与外部系统的集成（API、数据库、文件系统）
3. 优化 Agent 的推理效率和响应质量
4. 设计和实现 Agent 的记忆和上下文管理
5. 构建 Agent 的可观测性和监控系统

**任职要求**：
1. 3 年以上 Python 开发经验
2. 熟悉 LangChain/LlamaIndex 等 Agent 框架
3. 了解 RAG 技术和向量数据库
4. 有 SQL 数据库操作经验
5. 良好的代码质量和工程实践

**加分项**：
1. 有 Agent 应用上线经验
2. 了解 MCP 协议
3. 有开源项目贡献经验

### 3.2 智能体平台工程师（高级）

**岗位职责**：
1. 设计和构建 AI Agent 开发平台
2. 实现 Agent 的可观测性和监控系统
3. 优化 Agent 基础设施的性能和可靠性
4. 推动 Agent 技术标准化和最佳实践

**任职要求**：
1. 5 年以上后端开发经验
2. 精通 Python，熟悉 Go/Rust
3. 深入理解 Agent 架构和设计模式
4. 有大规模分布式系统经验
5. 熟悉云原生技术和容器编排

**加分项**：
1. 有 Agent 平台开发经验
2. 熟悉 OpenTelemetry 或类似监控系统
3. 有开源项目维护经验

### 3.3 LLM 应用工程师

**岗位职责**：
1. 基于大模型开发智能应用
2. 优化 Prompt 设计和模型调用
3. 实现 RAG 系统和知识库
4. 构建模型评估和优化流程

**任职要求**：
1. 3 年以上 Python 开发经验
2. 熟悉 OpenAI/Anthropic/智谱等大模型 API
3. 了解 Prompt Engineering 和 Fine-tuning
4. 有向量数据库使用经验
5. 良好的问题分析和解决能力

**加分项**：
1. 有 LLM 应用上线经验
2. 了解模型评估方法
3. 有 NLP 背景

## 四、技能需求矩阵

### 4.1 必须技能

| 技能 | 要求程度 | 使用场景 |

| Python | **必须** | Agent 开发、模型调用、数据处理 |
| LangChain/LlamaIndex | **必须** | Agent 编排、工具调用 |
| RAG 技术 | **必须** | 检索增强生成、向量数据库 |
| SQL | **必须** | 数据查询、数据库操作 |
| API 开发 | **必须** | RESTful API 设计和实现 |
| Git | **必须** | 版本控制、协作开发 |

| 学习能力 | **必须** | 技术快速迭代 |

## 五、与我们项目的关联

### 5.1 我们项目覆盖的 JD 需求

| JD 需求 | 我们的覆盖 | 差距 |
|---------|-----------|------|
| 多数据源集成 | ✅ SQL、向量、文件、API | 已完善 |
| Agent 应用开发 | ✅ AgentDataClient | 需要更多示例 |
| 向量数据库 | ✅ Chroma、InMemory | 需要 Qdrant/Pinecone |
| 数据管道 | 🔜 基础实现 | 需要完善 ETL 功能 |
| 可观测性 | ✅ OpenTelemetry | 已完善 |
| 缓存优化 | ✅ 上下文感知缓存 | 已完善 |

### 5.2 我们的差异化优势

| 维度 | 我们 | LangChain | LlamaIndex |
|------|------|-----------|------------|
| Agent 专注 | ✅ 专为 Agent 设计 | 通用 LLM 应用 | RAG 专注 |
| 统一接口 | ✅ 一致的 API | 分散的抽象 | 分散的抽象 |
| 上下文感知 | ✅ Agent 上下文 | 无 | 有限 |
| 可观测性 | ✅ 内置追踪 | 依赖 LangSmith | 有限 |
| 轻量级 | ✅ 轻量设计 | 较重 | 中等 |
